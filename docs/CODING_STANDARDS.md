# Coding Standards

## General

Readable over clever.

Small functions.

Reusable components.

No duplicated code.

## React

Functional components only.

Use hooks.

No prop drilling when avoidable.

Keep components focused.

## TypeScript

No any.

Use interfaces.

Strict typing.

## Styling

Tailwind only.

No inline styles.

Reuse utility classes.

## Backend

Constructor injection.

Service layer.

DTOs.

Validation.

Global exception handling.

No business logic in controllers.

## Naming

PascalCase

Components

camelCase

Variables

UPPER_CASE

Constants

kebab-case

Folders

## Git

One feature per branch.

One User Story per PR.

Small commits.

Meaningful commit messages.
